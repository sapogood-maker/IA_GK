## 🔄 ANTES vs DEPOIS

### ANTES (Problema)
```python
# ❌ Authorization era query parameter (inseguro)
async def get_current_user(authorization: str = None, db: AsyncSession = Depends(get_db)):
    # Código repetido em cada endpoint
    # 19 linhas de lógica de validação
```

**Problemas:**
- Token exposto em logs de URL
- Lógica duplicada em múltiplos endpoints
- Não segue padrão HTTP/REST (RFC 7235)
- Query parameter com default `None` (ambíguo)

### DEPOIS (Corrigido)
```python
# ✓ Authorization é header obrigatório (seguro)
async def get_current_user(authorization: str = Header(...), db: AsyncSession = Depends(get_db)):
    # Dependência centralizada e reutilizável
    # 27 linhas com lógica completa (1 vez apenas)

@router.get("/me")
async def me(user = Depends(get_current_user)):
    # Endpoint limpo e simples: 6 linhas
    return {...}
```

**Benefícios:**
- Token em header HTTP (seguro, RFC 7235)
- Lógica centralizada (sem duplicação)
- Reutilizável em outros endpoints
- Código limpo e legível
- Swagger auto-gerado corretamente

## 📝 FLUXO DE AUTENTICAÇÃO

```
1. LOGIN
   POST /api/v1/auth/login
   Body: { email, password }
   └─> Response: { access_token, refresh_token }

2. GET USER INFO (CORRIGIDO)
   GET /api/v1/auth/me
   Header: Authorization: Bearer {token}  ✓
   └─> Response: { id, name, email, role }

3. REFRESH TOKEN
   POST /api/v1/auth/refresh
   Body: { refresh_token }
   └─> Response: { access_token, refresh_token }
```
